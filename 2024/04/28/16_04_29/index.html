同步方案

[!abstract] 之前分享过以 macOS 为中心，使用群晖 Drive 和 iCloud 同步群晖和苹果生态中的 obsidian 库（详见 苹果全平台无缝同步群晖Obsidian库），但这个方案有一个致命缺点就是需要一个 Mac 作为中继，同步群晖和 iCloud。无法做到任意设备的实时同步。

[!Help] 这篇文章分享一个可以实时同步的方案顺便分享一些不错的 obsidian 第三方插件。

Remotely save

[!note] 一个非官方的 Obsidian 插件允许用户在本地设备和云服务之间同步笔记。支持 S 3、Dropbox、OneDrive、webdav。

按需求链接，这里以 webdav 为例，输入地址、用户名、密码进行连接。

可以加密同步，我是为了同步到群晖，自动 git push 到 GitHub pages，所以就没有选择加密。

还有其他功能设置，可自行按需设置。

Obsidian 第三方插件

Completr 自动联想补全插件

[!summary] 插件提供了 LaTex，Frontmatter，callout 的自动联想补全功能，还提供了扫描当前文件所有文字作为联想备选文字的功能。另外，你还可以自己提供放了文字的文件作为联想的数据。

  • 输入 >[ 就会出现所有 ob 提供的 callout，点击确认即可自动补全。

  • 提供字库作为联想备用文字

  • 自动补全 LaTex

简单配置

默认无法扫描到中文，如果需要中文联想，需要在设置的第一项 Word character regex 的开头插入：\u4e00-\u9fa5，这样就可以扫描到中文了

不要去掉原来的 a-zA-ZöäüÖÄÜß，那是用来匹配字母的。结果为 \u4e00-\u9fa5a-zA-ZöäüÖÄÜß；

插件提供的可以修改的设置有:

  1. 最小开始联想的字符；
  2. 替换的方式（是否忽略大小写，是直接替换还是结尾补全等）
  3. 是否启用 LaTex 补全（以及是否需要在代码块中提供补全，是否忽略大小写等）
  4. 是否启用 Frontmatter 补全（以及是否忽略大小写，是否添加后缀）
  5. 是否需要扫描文件作为补全的字库（以及是否包括当前文件）
  6. 是否需要我们自己提供文件作为字库；
  7. 是否启用 Callout 补全功能；
